# OrgMemory Project Context

OrgMemory is built around a narrow MVP: a registry for organizational AI
capability assets.

## Positioning

OrgMemory does not try to be a general wiki, a GoClaw-style agent runtime, or a
workflow builder. It manages the operational assets that emerge when employees
use AI well:

- prompts
- repeatable workflows
- agent configurations
- input/output schemas
- examples
- best practices
- ownership and handover context

## Source Brief

The original document from the teacher frames the product as Organizational AI
Capital Management. It warns that the real problem is not storage alone. The
real problem is contribution behavior: incentives, social contract, and habit.

Phase 1 should build the registry, not the capture system. Do not build passive
ChatGPT or Claude capture in the MVP.

## MVP Definition

The MVP is done when a demo user can:

1. Create a capability asset.
2. Add structured metadata: owner, use case, department, AI tool, tags.
3. Search and open the asset.
4. Click "Use this asset" and increment usage tracking.
5. See version history.
6. Move an asset through a basic review status.
7. See lightweight onboarding/offboarding panels that explain the future workflow.

Demo success metric: 20 prompts or workflows submitted and actively used by at
least 5 different people in the demo environment.
