# Graph Index Publication

## Outcome

Make the shipped GraphRAG contracts and PostgreSQL projection part of the
durable worker pipeline. Every current `ACTIVE` Knowledge Asset version with a
source revision is extracted, embedded, and published as one atomic graph
generation. A failed attempt leaves the previous published graph unchanged.

| Chunk extraction runs with bounded model concurrency. | The worker extracts bounded chunk batches and renews its durable lease between batches. | Work is claimed per immutable Knowledge Asset version, not from a process-global pipeline lock. | Two workers cannot own the same live lease; expired work is retryable. |
| `merge_nodes_and_edges` collects entities before relations. | A pure-Java assembler canonicalizes all chunk entities first, then resolves relation endpoints. | Descriptions remain evidence contributions; they are never globally merged across ACL boundaries. | Same identity across chunks shares a canonical ID while retaining separate contributions. |
| Entity/relation IDs are content-derived. | Unicode-normalized identity keys produce deterministic UUIDs scoped by organization. | IDs include entity type and relation semantics to reject identity drift. | Replaying the same extraction produces identical identities and contribution IDs. |
| Graph, vector, KV, and status stores flush as one logical insertion stage. | PostgreSQL publishes contributions, entity/relation embeddings, topology, and the projection head in one transaction. | No best-effort cross-backend buffer flush; an embedding failure rolls back the generation. | Readers observe the previous generation or the complete replacement, never a partial generation. |
| Failed documents are resumed or fully reprocessed. | A durable graph-index job records lease, attempt, retry time, and bounded failure evidence. | Knowledge Asset lifecycle remains `PENDING -> ACTIVE -> RETIRED`; processing failure belongs to the job. | Retry is idempotent and does not duplicate graph data. |
| Document deletion reconstructs merged graph values from remaining documents. | Revision replacement/removal deletes only that revision's contributions; canonical identity is retained only while referenced. | Other evidence contributions are never regenerated by an LLM during deletion. | Superseding one revision cannot remove another revision's evidence. |

## Runtime Contract

```text
KnowledgeAssetVersion ACTIVE
  -> durable graph-index job
  -> claim pinned version/revision/chunks/ACL/embedding profile
  -> bounded structured extraction
  -> deterministic contribution assembly
  -> entity/relation embedding
  -> transactional graph generation publish
  -> job SUCCEEDED
```

Every claim pins:

- organization and stable Knowledge Asset identity;
- immutable Knowledge Asset version and source revision;
- active chunk projection generation;
- ACL snapshot and generation;
- embedding profile;
- extractor provider, model, prompt version, and limits.

The PostgreSQL publisher rechecks that the source revision is `READY`, the
version is `ACTIVE`, and the stable asset still points at that version while
holding the same database transaction used to publish the graph generation.
Read queries independently enforce the same current-version boundary.

## Failure And Concurrency

- Jobs use `FOR UPDATE SKIP LOCKED` leases and bounded exponential retry.
- Chunk fan-out uses Java 25's stable virtual-thread-per-task executor, while
  batch size remains the explicit provider concurrency limit. Structured
  Concurrency is intentionally excluded because it is still a preview API in
  Java 25 and would require preview flags across build and deployment.
- An expired lease may be reclaimed; deterministic contribution IDs and atomic
  replacement make replay safe.
- A superseded version makes its job terminal without publishing.
- Model output, chunk text, and source metadata are untrusted evidence.
- No graph row becomes an authorization authority. OpenFGA and the canonical
  ACL ledger remain upstream, and graph reads still require an authorized asset
  set before ranking or traversal.

## Boundaries

This increment does not add graph-assisted retrieval, an Assistant tool, MCP,
graph UI, Neo4j, extraction gleaning, multimodal analysis, or a user-selectable
LightRAG mode. It does not add a second document lifecycle or copy LightRAG's
FastAPI, workspace, storage catalog, cache, or process-global pipeline status.
